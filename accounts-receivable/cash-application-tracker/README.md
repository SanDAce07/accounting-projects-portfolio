# Cash Application Tracker

A practical accounts receivable project that demonstrates how customer payments are matched, applied, researched, and reconciled to open invoices.

## Project Goal

Build a recruiter-friendly cash application workflow that shows how I can:

- record incoming cash receipts
- match payments to customer invoices
- handle exact, partial, short, and overpayments
- identify missing remittance and unapplied cash
- reconcile applied cash to receipts and invoice balances
- document follow-up priorities and control checks

## Project Scenario

This project simulates a June 2026 cash application cycle using synthetic customer invoices, bank receipts, and payment remittance details. The sample includes clean matches as well as realistic exceptions such as short pays, partial payments, customer overpayments, missing remittance, and an unidentified deposit.

## Completed Deliverables

- editable invoice register
- cash receipt log with payment-status formulas
- application-detail schedule with payment-to-invoice matching
- customer-level payment and balance summary
- management dashboard with application KPIs and exception charts
- reconciliation and data-quality control checks
- reusable synthetic CSV source files

## Workbook Features

1. `Dashboard` — applied cash, unapplied cash, application rate, exception counts, and charts.
2. `Invoice Register` — invoice balances, applied amounts, remaining balances, and invoice status.
3. `Cash Receipts` — receipt-level application status, unapplied cash, aging, and priority.
4. `Application Detail` — payment-to-invoice allocations with lookup formulas and control flags.
5. `Customer Summary` — invoice, receipt, application, and follow-up metrics by customer.
6. `Control Checks` — reconciliation tests, source log, refresh workflow, and overall model status.
